cmsimde

SMap - SiteMap - 依照階次列出網站的所有頁面.

EditA - Edit All page - 將所有頁面放入編輯模式中, 主要用來處理頁面搬遷, 刪除或決定衝突頁面內容版本.

Edit - Edit page - 先選擇要編輯的單一頁面後, 再點選 Edit, 即可進入單一頁面的編輯模式.

Config - Configure Site - 編輯頁面標題與管理者密碼.

Search - 動態頁面內容的關鍵字搜尋.

IUpload - Image file Upload - 圖檔的上傳功能, 可以上傳 jpg, png 與 gif 檔案, 其中若在手機上傳圖檔, 系統會自動將圖片檔案縮小為 800x800 大小.

IList - Image file List - 列出可以直接在頁面編輯模式中引用的圖片檔案.

FUpload - File Upload - 一般檔案的上傳功能, 目前可以上傳的檔案副檔名包括 'jpg', 'png', 'gif', '7z', 'pdf', 'zip', 'ttt', 'stl', 'txt', 'html', 'mp4' 等, 使用者可以自行修改.

FList - File List - 列出可以直接在頁面編輯模式中引用的上傳檔案.

Logout - 登出頁面編輯模式.

Convert - 將動態網站中位於 config/content.htm 檔案, 透過分頁設定轉為 content 目錄中的靜態網頁.

acp - git add, git commit 與 git push, 通常只有在 localhost 或自架主機上利用網頁表單協助將倉儲改版內容推向 Github 倉儲.

SStatic - Start Static Site - 利用 Python 啟動網站伺服功能, 可以讓使用者檢查轉檔後的靜態網站內容.

RStatic - Replit Static Site URL - 僅用於 Replit 模式, 當使用者按下 SStatic 後, 可以按下 RStatic 進入當下尚未推向 Github Pages 的靜態網站.

80 - 由 init.py 中 static_port 所決定的連結字串, 一般不使用 80, 只有在 Replit 中為了與動態網站共用 port, 才特別設為 80.

網頁式內容管理-學習議題





1.如何建立 Github 上的網站?

  1. 创建一个仓库： 在GitHub上创建一个新的仓库，按照前面提到的步骤，命名为“username.github.io”。

  2. 上传网站文件： 将您的网站文件上传到该仓库中。这些文件可以是HTML、CSS、JavaScript等。

  3. 启用GitHub Pages： 在仓库设置中选择要用作GitHub Pages的源，并启用GitHub Pages服务。

  4. 等待部署完成： GitHub Pages会自动构建和部署您的网站，等待部署完成。

  5. 访问您的网站： 一旦部署完成，您可以通过访问“https://username.github.io”来查看您的网站。


2.如何管理內容?

  • 更新文件： 每当需要更新网站内容时，只需将新的文件或更新上传到GitHub仓库中。
  • 版本控制： 使用Git进行版本控制，跟踪更改并回滚到需要的版本。
  • 分支管理： 使用Git分支进行开发和测试，然后将更改合并到主分支中。

3.如何建立章節式網站?

  • 使用HTML和CSS来创建不同章节的页面。
  • 使用导航栏或侧边栏来导航到不同的章节。

4.能放入甚麼多媒體內容?

  • 静态图文： 在HTML中插入图片和文字。
  • 动态图文： 使用JavaScript或CSS动画来创建动态效果。
  • 在线游戏： 将游戏嵌入到网页中，可以使用HTML5 Canvas或WebGL等技术。
  • 3D零件展示： 使用JavaScript库（如Three.js）创建和展示3D模型。

5.靜態圖文?

  1. 创建一个仓库： 在GitHub上创建一个新的仓库，用于托管您的静态图文内容。

  2. 编写HTML和CSS： 使用您喜欢的文本编辑器编写HTML和CSS代码来创建您的静态页面。在HTML中，您可以使用标签来定义文本内容和图像，并使用CSS来样式化页面。

  3. 上传文件： 将您编写的HTML和CSS文件上传到GitHub仓库中。确保所有的图片和其他媒体文件也一并上传到仓库中。

  4. 启用GitHub Pages： 在仓库的设置页面中，选择要用作GitHub Pages的源，并启用GitHub Pages服务。这样，您的静态网页将可以通过GitHub Pages进行托管，并可以通过您的GitHub Pages域名访问。

  5. 等待部署完成： GitHub Pages会自动构建和部署您的静态页面，等待部署完成。

  6. 访问您的静态页面： 一旦部署完成，您可以通过访问您的GitHub Pages域名（通常是“https://username.github.io/repositoryname”）来查看您的静态页面。

6.動態圖文?

  1. 创建一个仓库： 在GitHub上创建一个新的仓库，用于托管您的动态图文内容。

  2. 编写HTML、CSS和JavaScript：

    • HTML：定义页面结构和内容。
    • CSS：样式化页面元素，使其看起来更加美观。
    • JavaScript：添加交互式功能和动态效果。

  3. 上传文件： 将您编写的HTML、CSS和JavaScript文件以及其他必要的资源（如图片、音频、视频等）上传到GitHub仓库中。

  4. 启用GitHub Pages： 在仓库的设置页面中，选择要用作GitHub Pages的源，并启用GitHub Pages服务。

  5. 等待部署完成： GitHub Pages会自动构建和部署您的动态页面，等待部署完成。

  6. 访问您的动态页面： 一旦部署完成，您可以通过访问您的GitHub Pages域名来查看您的动态页面。

7.線上遊戲?

  1. 创建一个仓库： 在GitHub上创建一个新的仓库，用于托管您的在线游戏内容。

  2. 编写HTML、CSS和JavaScript：

    • HTML：定义游戏的结构和布局。
    • CSS：样式化游戏元素，使其看起来更加美观。
    • JavaScript：实现游戏的逻辑和交互功能。

  3. 上传文件： 将您编写的HTML、CSS和JavaScript文件以及游戏所需的其他资源（如图像、音频、视频等）上传到GitHub仓库中。

  4. 启用GitHub Pages： 在仓库的设置页面中，选择要用作GitHub Pages的源，并启用GitHub Pages服务。

  5. 等待部署完成： GitHub Pages会自动构建和部署您的游戏页面，等待部署完成。

  6. 访问您的游戏页面： 一旦部署完成，您可以通过访问您的GitHub Pages域名来玩游戏。

8.3D 零組件展示?

  1. 创建一个仓库： 在GitHub上创建一个新的仓库，用于托管您的3D零组件展示内容。

  2. 编写HTML、CSS和JavaScript：

    • HTML：定义包含3D场景的页面结构。
    • CSS：为页面元素添加样式。
    • JavaScript：使用WebGL或其他3D库（如Three.js）来创建和渲染3D场景。

  3. 上传文件： 将您编写的HTML、CSS和JavaScript文件以及展示所需的其他资源（如3D模型文件、纹理、音频等）上传到GitHub仓库中。

  4. 启用GitHub Pages： 在仓库的设置页面中，选择要用作GitHub Pages的源，并启用GitHub Pages服务。

  5. 等待部署完成： GitHub Pages会自动构建和部署您的展示页面，等待部署完成。

  6. 访问您的展示页面： 一旦部署完成，您可以通过访问您的GitHub Pages域名来查看您的3D零组件展示。

9.如何建立網頁系統中的網誌?

  • 使用静态网页生成器（如Jekyll、Hugo等）创建博客网站。
  • 将博客文章以Markdown格式编写，然后生成HTML页面。

10.如何建立網頁系統中的簡報?

  • 使用HTML和CSS创建简报页面。
  • 使用JavaScript库（如reveal.js）来提供简报的交互性和动画效果。

git 的發展歷史

Git 是一個分散式版本控制系統，由 Linus Torvalds 於 2005 年創建。Git 的發展歷史可以追溯到 Linux 內核開發過程中的需求。以下是 Git 的發展歷程：

  1. 2005 年：Linus Torvalds 開始開發 Git。他尋找了一個更好的版本控制系統來管理 Linux 內核的開發，因為 BitKeeper 不再免費提供使用。

  2. 2005 年 4 月：Git 的第一個版本（1.0）發布。在開發過程中，許多 Linux 開發者貢獻了代碼，幫助 Git 的成長。

  3. 2005 年至今：Git 穩步成長，吸引了全球開發者社區的關注和貢獻。它迅速成為了最受歡迎的版本控制系統之一，廣泛應用於軟體開發領域。

  4. 2008 年：GitHub 成立，為 Git 提供了一個流行的主機服務平台，使得開發者可以輕鬆共享和協作。

  5. 2016 年：Microsoft 買下 GitHub，這一舉動在開源社區引起了一些爭議，但也為 Git 的發展提供了更多資源和支持。

  6. 2020 年：GitHub 宣布啟動代碼移動計劃，計劃將所有項目的主要代碼存儲庫轉移到遠程儲存庫中。這項計劃將加速代碼庫的存取速度，提高性能。

總的來說，Git 的發展歷史是一個開放和協作的過程，得益於全球開發者社區的不懈努力和貢獻。Git 不僅在軟體開發中得到廣泛應用，還成為了許多領域進行版本控制的首選工具。

git創作人的相關履歷

Git 的創造者是 Linux 核心的創建者 Linus Torvalds。他是一位芬蘭籍的計算機科學家，於 1991 年創建了 Linux 作業系統，並在開源社區中享有極高的聲譽。

Linus Torvalds 出生於 1969 年 12 月 28 日，畢業於赫爾辛基大學，主修電腦科學。他在創建 Linux 核心的同時，也是 Git 版本控制系統的創建者和主要開發者。

在他的職業生涯中，Torvalds 曾就職於 Transmeta Corporation 和 Open Source Development Labs。他的工作和貢獻使他成為了開源社區的一個重要人物，並且獲得了多項獎項和榮譽。

Torvalds 以其開放、直率和自由的風格而聞名，他在維護 Linux 核心和 Git 項目的過程中始終堅持這一風格。他通過他的工作，為開源軟體的發展和普及做出了重大貢獻。

請問git如何應用在Master CAM

Git 在機械設計專業中的應用通常不是直接與 Master CAM（一種用於電腦輔助製造的軟體）整合，而是在開發流程中用於版本控制和協作。以下是一些在機械設計專業中使用 Git 的情況：

  1. 版本控制：機械設計過程中會產生大量的 CAD（電腦輔助設計）文件、製造程序等。使用 Git 可以輕鬆管理這些文件的版本，追蹤更改歷史，並能夠在需要時回復到先前的版本。這使得團隊成員可以同步工作，而不必擔心文件的覆蓋或遺失。

  2. 協作：Git 可以使得團隊成員能夠同時進行機械設計項目的協作。他們可以在本地工作，然後通過 Git 將他們的更改推送到共享的遠程存儲庫中。這樣就可以避免文件的碰撞，並確保所有人都在同一頁面上工作。

  3. 分支管理：Git 的分支功能使得可以輕鬆地創建新的功能分支或修復分支，而不會影響到主要的開發分支。這對於在機械設計項目中進行不同版本或實驗性工作非常有用。

  4. 代碼審查：即使在機械設計中，代碼審查也是一個重要的步驟。Git 的分支和合併功能使得可以輕鬆地進行代碼審查，確保代碼的質量和一致性。

總的來說，雖然 Git 不是直接與 Master CAM 整合的工具，但它可以作為機械設計專業中的一個強大的版本控制和協作工具，幫助團隊更有效地進行工作。

亦即將所有對應的 html 也使用 Brython 產生, 然後寫為 class 後, 在範例導入時透過 instance 引用.

<!-- 啟動 Brython -->
<script>
window.onload=function(){
brython({debug:1, pythonpath:['/static/','./../downloads/py/']});
}
</script>
